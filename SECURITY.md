SECURITY.md
============

Key management and security guidance for Twin Life Enterprise PoC

- Secrets storage: Use Azure Key Vault (`twinlife-vault`) to store all runtime
  secrets. Example secret names:
  - `TwinLife-ClientSecret` — Entra app client secret (for CI / demo automation)
  - `SIM_API_KEY` — optional demo API token for the local microservice (avoid in commits)

- Permissions:
  - Key Vault is configured to use RBAC. Grant the runtime identity one of:
    - `Key Vault Secrets Officer` (minimal for read/write secrets)
    - `Key Vault Contributor` / `Key Vault Administrator` (broader management)
  - For automation prefer a service principal or managed identity; give the
    principal the vault-scoped role. Do NOT publish credentials in the repo.

- Rotation & lifecycle:
  - Rotate client secrets regularly (e.g. annually for demos; more frequently for production).
  - After creating a service principal, store its credentials securely (not in git) and rotate them.

- Local development:
  - For quick local runs you may set `SIM_API_KEY` in your environment. NEVER commit this value.
  - Prefer to sign into Azure CLI with an AAD user for admin tasks: `az login --use-device-code`.

- Audit & verification:
  - We include `audit.json` generated by `scripts/generate_summary.js` in release artifacts.
  - Before submission verify no secret values are committed: `git grep -n "TwinLife-ClientSecret\|SIM_API_KEY"`.

- Contact & escalation:
  - If you need RBAC changes ask your subscription Owner or User Access Administrator to run the role assignment commands.

Example commands (Owner required to assign role):
```bash
# vault id
VAULT_ID=$(az keyvault show --name twinlife-vault --query id -o tsv)
# assign role to service principal/object id
az role assignment create --assignee-object-id <OBJECT_ID> --role "Key Vault Secrets Officer" --scope "$VAULT_ID"
# set secret (after role assignment propagates)
az keyvault secret set --vault-name twinlife-vault --name TwinLife-ClientSecret --value "<secret>"
```

Do not include secret values or `sp.json` in the repository.
