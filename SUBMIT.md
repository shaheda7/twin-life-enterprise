Twin Life Enterprise — Quick Run for Judges
=========================================

Prerequisites
- Node.js (v16+), `gh` CLI (optional), `unzip` or `tar`, network access to localhost.

Download the submission ZIP (recommended: `gh` CLI)

1) Using `gh` (recommended):

```bash
gh release download v1.0 --repo shaheda7/twin-life-enterprise --pattern "submission_summary.zip" --dir .
```

2) If you prefer `curl` (open release page and copy the exact asset URL):

```bash
# Example (open release page to get exact URL if this fails):
curl -L -o submission_summary.zip "https://github.com/shaheda7/twin-life-enterprise/releases/download/v1.0/submission_summary.zip"
```

Unpack and run

```bash
# Unzip into ./submission
unzip submission_summary.zip -d submission
cd submission

# Start the microservice (background)
cd microservice
SIM_API_KEY=devkey node index.js &

# (optional) check health
curl -sS -H "Authorization: Bearer devkey" http://localhost:5001/health

# In a new terminal, run the summary generator (from the submission root)
cd ..
SIM_API_KEY=devkey node scripts/generate_summary.js
```

What judges should see
- `SUMMARY.html` and `SUMMARY.pdf` generated in the submission root.
- `audit.json` containing decision trace and inputs.
- `microservice/submission_summary.zip` (packaged artifact) should be downloadable from the release.

Quick smoke tests

```bash
# Compare two sample scenarios directly
curl -sS -X POST http://localhost:5001/compare \
  -H "Authorization: Bearer devkey" \
  -H "Content-Type: application/json" \
  -d '{"scenarioA": {"name":"A","workload":1000}, "scenarioB": {"name":"B","workload":1200}}' | jq

# Fetch stored results (replace <id> with returned id)
curl -sS -H "Authorization: Bearer devkey" http://localhost:5001/results/<id> | jq
```

Cleanup

```bash
# Kill the microservice started in background (macOS / Linux)
pkill -f "node index.js" || true
```

Notes for judges
- The service listens on port `5001` by default. If port conflicts occur, update `PORT` env var before starting.
- For local testing we use `SIM_API_KEY=devkey` as a demo API key; do NOT use real secrets in the repo.

Troubleshooting
- If `gh release download` fails, open the release page and download `submission_summary.zip` manually:
  https://github.com/shaheda7/twin-life-enterprise/releases/tag/v1.0
